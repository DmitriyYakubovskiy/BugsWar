# Bugs War

**Bugs War** — прототип трёхмерной стратегии в реальном времени, разработанный на Unity.

Игрок управляет армией боевых насекомых: выбирает доступных юнитов, размещает их на поле и расходует ограниченный ресурс. Противник автоматически создаёт собственные отряды, после чего юниты самостоятельно перемещаются к цели и вступают в бой.

## Об игре

В Bugs War сражаются две армии насекомых.

Основная задача игрока — правильно распределять ресурсы, выбирать подходящих бойцов и размещать их на поле так, чтобы остановить наступление противника.

После размещения юниты действуют автоматически:

- двигаются по игровому полю;
- обнаруживают противников;
- приближаются к ним;
- атакуют;
- получают урон;
- уничтожаются при потере здоровья.

По мере развития матча противник становится опаснее: изменяется частота появления его бойцов и могут появляться усиленные юниты.

Основной игровой цикл выглядит следующим образом:

```text
Получение ресурса
       │
       ▼
Выбор типа юнита
       │
       ▼
Размещение на поле
       │
       ▼
Списание стоимости
       │
       ▼
Автоматическое движение
       │

Сражение
```

## Управление

Основное управление выполняется мышью.

### Размещение юнита

1. Выберите юнита на панели интерфейса.
2. Наведите курсор на доступную область игрового поля.
3. Нажмите левую кнопку мыши.
4. При наличии достаточного количества ресурса юнит появится на выбранной позиции.

Перед созданием система проверяет:

- выбран ли юнит;
- не находится ли курсор над интерфейсом;
- достаточно ли ресурса;
- попадает ли луч камеры в игровое поле.

## Ресурс снабжения

Для создания бойцов используется ресурс снабжения.

У каждого типа юнита есть собственная стоимость:

```text
Текущее снабжение ≥ Стоимость юнита
```

При недостатке ресурса размещение не выполняется.

Система снабжения ограничивает количество одновременно создаваемых бойцов и заставляет игрока выбирать между дешёвыми и более сильными юнитами.

## Юниты

Каждый юнит может иметь следующие параметры:

| Параметр | Назначение |
|---|---|
| `Lives` | Запас здоровья |
| `Cost` | Стоимость создания |
| `Damage` | Урон за одну атаку |
| `AttackInterval` | Интервал между атаками |
| `Speed` | Скорость передвижения |
| `Tag` | Принадлежность к команде |

## Типы юнитов

В проекте используются разные префабы бойцов.

Среди них присутствуют как минимум:

- стандартный юнит;
- более тяжёлый или танковый юнит;
- усиленный крупный противник.

Разные бойцы могут отличаться:

- стоимостью;
- здоровьем;
- уроном;
- скоростью;
- размером;
- частотой атак.

## Команды

При создании юнит получает тег своей стороны.

Упрощённо игровая логика разделяет объекты на две команды:

```text
Игрок
  │
  ├── Юнит 1
  ├── Юнит 2
  └── Юнит 3

Противник
  │
  ├── Юнит 1
  ├── Юнит 2
  └── Усиленный юнит
```

Тег используется для определения союзников и противников.

## Автоматический противник

Противник управляется системой автоматического спавна.

Она:

1. хранит набор доступных префабов;
2. выбирает случайный тип юнита;
3. рассчитывает случайную позицию в области появления;
4. создаёт объект;
5. назначает ему тег команды;
6. ожидает заданный интервал;
7. повторяет процесс.

Пример логики:

```text
Ожидание
   │
   ▼
Выбор случайного префаба
   │
   ▼
Выбор позиции
   │
   ▼
Создание юнита
   │
   ▼
Назначение команды
   │
   └────────── повтор
```

## Сложность

Сложность матча изменяется в зависимости от набранных очков.

Возможные изменения:

- сокращение интервала появления противников;
- остановка стандартного спавна;
- создание специального усиленного юнита;
- увеличение его размера;
- увеличение здоровья;
- изменение скорости;
- изменение урона.

Это позволяет строить матч как последовательность этапов.

## Усиленный противник

На определённом этапе может появиться крупный противник с особыми характеристиками.

Для него могут изменяться:

- масштаб объекта;
- скорость `NavMeshAgent`;
- количество здоровья;
- сила атаки;
- тег команды.

Такой юнит выполняет роль босса или финального испытания.

## Боевая система

За нанесение урона отвечает отдельный контроллер атаки.

Типичная последовательность:

```text
Обнаружение цели
       │
       ▼
Проверка дистанции
       │
       ▼
Ожидание интервала атаки
       │
       ▼
Нанесение урона
       │
       ▼
Проверка здоровья цели
       │
       ▼
Уничтожение цели
```

Основные характеристики боя:

- урон;
- скорость атаки;
- дистанция;
- здоровье цели.

## Передвижение

Для перемещения используется навигационная система Unity.

Компонент `NavMeshAgent` позволяет юнитам:

- строить маршрут;
- обходить препятствия;
- двигаться к выбранной цели;
- изменять скорость;
- останавливаться возле противника.

## Физика

- `Rigidbody`;

Для отдельных юнитов настраиваются:

## Интерфейс

Пользовательский интерфейс позволяет:

- выбирать тип бойца;
- видеть доступное снабжение;

- взаимодействовать с кнопками размещения;
- отличать выбранный тип юнита.

При клике по игровому полю выполняется проверка, что курсор не находится над UI.

## Основные скрипты

Проект может включать следующие игровые компоненты:

### `Unit`

Хранит данные юнита:

- здоровье;
- стоимость;
- принадлежность к команде;
- дополнительные параметры.

### `AttackController`

Управляет атакой:

- выбирает цель;
- контролирует интервал;
- наносит урон.

### `RandomSpawnUnitsManager`

Отвечает за автоматическое появление противников:

- хранит массив префабов;
- выбирает случайный объект;
- рассчитывает позицию;
- назначает тег;
- управляет интервалом появления.

### `SupplyController`

Управляет ресурсом игрока:

- хранит текущее значение;
- проверяет возможность покупки;
- уменьшает количество ресурса.

### Контроллер размещения

Отвечает за:

- выбор префаба;
- подсветку кнопки;
- обработку клика;
- проверку UI;
- создание юнита на поле.

## Установка

Клонируйте репозиторий:

```bash
git clone https://github.com/DmitriyYakubovskiy/BugsWar.git
```

Перейдите в папку:

```bash
cd BugsWar
```

Откройте Unity Hub.

Выберите:

```text
Add project from disk
```

Укажите корневую директорию репозитория.

После импорта откройте основную сцену из каталога:

```text
Assets/Scenes/
```

## Запуск в редакторе

1. Откройте проект в Unity.
2. Дождитесь импорта ресурсов.
3. Откройте сцену `Map_1`.
4. Нажмите кнопку `Play`.

## Создание сборки

```text

```

```text
Build
```

## Возможные проблемы

### Юниты не двигаются

Проверьте:

- создан ли NavMesh;
- находится ли юнит на NavMesh;
- добавлен ли `NavMeshAgent`;
- назначена ли цель;
- не установлена ли нулевая скорость.

### Юниты не атакуют

Проверьте:

- правильность тегов;
- наличие коллайдеров;
- радиус обнаружения;
- компонент `AttackController`;
- значение урона;
- интервал атаки.

### Юнит не размещается

Проверьте:

- выбран ли префаб;
- достаточно ли снабжения;
- назначена ли игровая поверхность;
- попадает ли луч камеры в коллайдер;
- не находится ли курсор над интерфейсом.

### Противники не появляются

Проверьте:

- массив префабов;
- значение интервала;
- активность объекта Spawn Manager;
- область спавна;
- запуск корутины;
- отсутствие ошибок в Console.
